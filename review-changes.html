Overview
What Changed and Why
The previous extraction guidance was too vague—it said "check the CSS" without showing exactly how to pull precise values. These changes make extraction deterministic and faithful.

● Precise Typography

Added size, line_height, and letter_spacing sub-fields to the typography schema. No more "tight tracking"—now it's "-0.02em".

● Imagery Classification

New imagery section captures visual style (photographic, illustrated, sketch, etc.), image treatment, and common subjects.

● CSS Extraction Guide

Complete rewrite of from-website.md with a step-by-step guide: CSS variables first, then computed styles, font loading detection.

● Color Precision

Checklist now maps specific CSS properties to specific output fields. Background color from body background-color, not guessing.

Sub-field Reference Table New

Sub-fieldTypeRequiredDescription
familystringyesFont family name with fallbacks (e.g., "Inter, system-ui, sans-serif")
weightstringyesFont weight — numeric ("600") or named ("bold")
stylestringyesDescriptive style notes (case, tracking description, etc.)
sizestringno (new)Reference font size with unit (e.g., "48px", "3rem")
line_heightstringno (new)Line height as unitless ratio or with unit (e.g., "1.2", "56px")
letter_spacingstringno (new)Letter spacing / tracking (e.g., "-0.02em", "0.05em", "normal")
Key design decision: These are optional fields (not required) to maintain backward compatibility. Existing visual-style.md files remain valid. But the extractor is now strongly guided to always capture them.

Workflow: Before vs After Changed

Before (6 steps, vague)
1. Receive URL
2. Fetch the page — Use web fetch to get the HTML/CSS
3. Take screenshots
4. Analyze — Identify colors, typography, layout, motion, mood
5. Generate — Output complete visual-style.md
6. Validate — Ensure all required fields are present
After (8 steps, specific)
1. Receive URL
2. Fetch the page — full HTML source
3. Extract CSS data — Parse inline styles, <style> blocks,
   linked stylesheets. CSS custom properties first.
4. Take screenshots — cross-reference extracted values
5. Analyze deeply — apply extraction checklist
6. Classify imagery — visual style of images/graphics
7. Generate — with exact values
8. Validate — hex codes, px/em/rem units, font names

6-Step Extraction Prompt Enhanced

Replaces the old "REQUIRED / RECOMMENDED" field list with a structured process:

STEP 1 — EXTRACT RAW CSS VALUES
Search the page source for:
- CSS custom properties (:root variables)
- @font-face declarations and Google Fonts links
- Key element styles: body background-color, body color, h1/h2/h3, buttons, links

STEP 2 — MAP TO COLORS
- colors.primary: page background color AND primary text color (minimum 2)
- colors.accent: CTA/button backgrounds, link colors, hover states
- colors.neutral: secondary text, borders, muted backgrounds
Every hex value must come directly from the CSS. Do not approximate.

STEP 3 — MAP TO TYPOGRAPHY
For each tier (display, body, caption), record:
- family, weight, style, size, line_height, letter_spacing
Capture the full type scale in typography.rules.

STEP 4 — CLASSIFY IMAGERY
- Visual style? (photographic, illustrated, sketch, geometric, etc.)
- Image treatment? (filters, shapes, borders, shadows, overlays)
- Subjects? (people, products, screenshots, abstract, patterns)

STEP 5 — CAPTURE MOTION & ANIMATION
- transition properties and durations
- @keyframes animation names and behaviors
- Scroll-triggered effects, hover effects

STEP 6 — GENERATE OUTPUT
Complete YAML frontmatter + Markdown body sections.
Include every value with its exact CSS source.

Enhanced Tips Enhanced

Went from 5 generic tips to 9 specific, actionable tips:

- CSS variables first — search for :root { or [data-theme] blocks
- Font loading tags — look for Google Fonts <link> or @font-face
- Computed values matter — record actual px, em, rem values
- Background ≠ white by default — verify the actual background-color
    Many sites use off-whites like #FAFAFA, #F8F9FA, #FFFDF5
- Heading color ≠ body color — check <h1> and <p> separately
- Letter-spacing is often negative on display text (-0.01em to -0.04em)
- Classify the imagery — photography? illustration? 3D renders?
- Note what's absent — no stock photos = deliberate choice → mood.avoid
- Capture the feel — weave colors, typography, imagery, motion together
